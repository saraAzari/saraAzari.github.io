IPC Dashboard - Data Visualization Libraries

Skip to end of metadata
  • Created by Nirmal Kanagasabai, last modified about 14 hours ago
Go to start of metadata

In order to implement charts on the Infection Prevention and Control (IPC) Dashboard, we decided to explore different data visualization libraries that might be suitable for the project.

Criteria

The following were the criteria used to choose / eliminate the charting libraries:

  • There should be out-of-the-box support for wide range of diverse chart types (Line, Bar, Pie, etc.)
  • They must be highly customizable to be able to achieve all (or most of) the UX requirements
  • Their APIs should be straight-forward and easy-to-use
  • There shouldn't be a steep learning curve to start using the library.
  • It should have concrete documentation with good examples. 
  • The library should be popular with an active community that continues to improve it.
  • Preferably, an open-source library that avoids licensing and other issues.

It is to be noted that IPC was entirely built using React which made us narrow down the libraries based on their compatibility.

Based on the criteria above, we came up with 4 charting libraries, namely:

  • React-Vis
  • Nivo
  • Highcharts
  • Recharts


We also looked at Chart.js and Victory charts. But, they were not as popular as the aforementioned data visualization libraries. If the project demands a web as well as mobile variant, 'Victory' charts could be a viable option. 

1. React-Vis

  • This library was developed by Uber and was designed with a modular and simplistic approach.
  • Their APIs are very straight forward and easy-to-use. 
  • React-Vis does not support responsiveness out-of-the-box. It requires an additional wrapper to achieve the same.

Vis - Financial Insights

It is to be noted that the 'Financial Insights' project used this library to build their data visualizations. We had a quick discussion with the developers involved. They expressed concerns about investing a lot of time and effort in customizing the components. 

Vis - Deprecated Library

Another downside is that the library has no active maintainers and is deprecated. 

2. Nivo

  • Nivo.rocks is built with D3.js. 
  • Out of all charting libraries, Nivo offers the best documentation. They use Storybook for UI demonstrations and examples. 
  • They also have a very active community. Unfortunately, their community is currently active in Discord and not in StackOverFlow. 
  • It has native support for themes and responsive components. 

3. Highcharts

  • They use pure JavaScript and can be used in React app. 
  • Highcharts is one of the most popular and comprehensive charting libraries that offer well-defined APIs, good set of examples and a terrific documentation. 
  • HighCharts has a full suite of multiple solutions - Highchart JS, Highstock JS, Highcharts Gantt and Highmaps JS. For our requirements, we may need Highcharts and Highstocks. 

Highcharts - License

Though HighCharts offer a free version for personal / non-commercial use, we need an organization-wide license to use the charting library. We did reach out to the HighCharts Sales team. Considering the urgency of the IPC project, we couldn't sort out the licensing details and decided to go with an open-source equivalent. 

4. Recharts

  • Recharts was built with D3.js taking 'modularity' and 'simplicity' into consideration.
  • Similar to Highcharts, Recharts is a very popular and comprehensive charting library with decent API support and a very active community. 
  • They are built on React components and it is interesting to note that they reuse their components to create composable components. 

Decision

Taking all the above-mentioned factors into consideration, we decided to eliminate React-Vis as the library was deprecated. Between Nivo and Recharts, Recharts had more popularity and an active community thereby eliminating Nivo. We narrowed down to two of the best solutions, 'Highcharts' and 'Recharts'. As we weren't able to sort out the licensing details for Highcharts, we decided to go with the Recharts, an open-source equivalent which can achieve almost everything that Highcharts does with minor compromises. 

Challenges encountered while using Recharts as the main data visualization library:

Recharts, the open-source library, although very helpful and easy-to-learn, did come with some compromises. The following were some of the challenges encountered while using Recharts as the main data visualization library:

1) Need to create custom components:

While implementing the project, we had to create some customized React components to meet the UX requirements. Some of those examples include:

  • Custom Y Axis Labels: To handle markdowns and italicize the words
  • Horizontal Scrollbar: As there wasn't any support for vertical scrollbar, we had to go with an horizontal scrollbar which was just a customized version of the 'Brush' component. 
  • Custom X-Axis Tick Formatter: We created a custom x-Axis Tick Formatter to format the date values
  • Custom Tooltip Component: Recharts library provides a Tooltip component out-of-the-box. However, they weren't sophisticated enough. To meet the UX requirements with legends on tooltips, we had to create a custom tooltip component to achieve the same. 

2) Need for custom data formatter:

While the library could natively handle input data for simple charts, for complex charts like, 'Stacked Bar Charts' and 'Line Charts involving Time-series data', we had to come up with two different data formatters. The Line Chart expected the data to be formatted. If not, they would not be plotting it linearly. 

Conclusion

While there are some challenges with Recharts, it is still a solid library built using React components. The library is actively being maintained by a big list of developers and Recharts enthusiasts.


To summarize:

'If the developers who are working on the project are comfortable with React, they can quickly create custom components that aren't currently present in Recharts library.'

'If there is a possibility to avoid the additional development effort on customizing / improving the tool, Highcharts, a commercial product, would be the best solution available in the market right now.'
